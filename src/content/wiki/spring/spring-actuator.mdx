## 정의

**Spring Boot Actuator**는 프로덕션용 monitoring/management endpoint 자동 제공. health check, metrics, env, beans, mappings 등. Micrometer로 Prometheus, CloudWatch, Datadog 등 다양한 모니터링 백엔드.

## 설정

```kotlin
implementation("org.springframework.boot:spring-boot-starter-actuator")
```

기본 노출: `/actuator/health`, `/actuator/info`. 다른 endpoint는 명시:

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health, info, metrics, prometheus
  endpoint:
    health:
      show-details: when-authorized    # always | never | when-authorized
```

### 커스텀 HealthIndicator

```java
@Component
public class ExternalApiHealthIndicator implements HealthIndicator {

    private final WebClient client;

    @Override
    public Health health() {
        try {
            String response = client.get().uri("/ping").retrieve().bodyToMono(String.class).block(Duration.ofSeconds(3));
            return Health.up().withDetail("response", response).build();
        } catch (Exception e) {
            return Health.down(e).build();
        }
    }
}
```

`/actuator/health/externalApi`로 개별 조회.

### Liveness vs Readiness

Kubernetes probe.

```yaml
management:
  endpoint:
    health:
      probes:
        enabled: true
      group:
        liveness:
          include: livenessState
        readiness:
          include: readinessState, db, redis
```

```
GET /actuator/health/liveness    # pod 살아있는지
GET /actuator/health/readiness   # 트래픽 받을 준비됐는지
```

## Micrometer

```java
@Service
public class OrderService {
    private final Counter counter;
    private final Timer timer;

    public OrderService(MeterRegistry registry) {
        this.counter = registry.counter("orders.created");
        this.timer = registry.timer("orders.processing.time");
    }

    public Order create(...) {
        return timer.recordCallable(() -> {
            counter.increment();
            return doCreate(...);
        });
    }
}
```

또는 어노테이션:

```java
@Timed("orders.processing.time")
@Counted("orders.created")
public Order create(...) { ... }
```

`@EnableAspectJAutoProxy`와 `aop` 의존성 필요.

## 보안

기본은 모든 endpoint 인증 필요. health, info만 노출.

```java
@Configuration
public class ActuatorSecurityConfig {

    @Bean
    public SecurityFilterChain actuatorSecurity(HttpSecurity http) throws Exception {
        http
            .securityMatcher(EndpointRequest.toAnyEndpoint())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(EndpointRequest.to("health", "info")).permitAll()
                .anyRequest().hasRole("ACTUATOR")
            )
            .httpBasic(Customizer.withDefaults());
        return http.build();
    }
}
```

별도 포트:

```yaml
management:
  server:
    port: 8081    # 메인 8080, actuator 8081
    address: 127.0.0.1
```

내부 네트워크에만 노출.

### 비즈니스 메트릭

```java
@Service
public class CheckoutService {
    private final Counter checkouts;
    private final Counter errors;
    private final DistributionSummary revenue;

    public CheckoutService(MeterRegistry registry) {
        this.checkouts = Counter.builder("business.checkouts").register(registry);
        this.errors = Counter.builder("business.checkouts.errors").register(registry);
        this.revenue = DistributionSummary.builder("business.revenue").register(registry);
    }

    public void checkout(Order order) {
        try {
            processOrder(order);
            checkouts.increment();
            revenue.record(order.total().doubleValue());
        } catch (Exception e) {
            errors.increment();
            throw e;
        }
    }
}
```

비즈니스 KPI를 대시보드에 자동.

## 함정

### 1. 민감 정보 노출

```bash
curl /actuator/env
# DB password, API key 등 노출 가능
```

show-values: `never` 또는 인증 + sanitize.

### 2. heapdump 큰 응답

GB 단위. timeout, 디스크 공간 주의.

### 3. metrics cardinality

```java
counter.increment(Tags.of("user_id", String.valueOf(userId)));    // 폭발!
```

태그 값이 unbounded면 metrics 수 폭발 → 메모리/저장 비용.

### 4. /actuator/threaddump의 성능 영향

stop-the-world 잠시. 자주 호출 X.

### 5. probe와 graceful shutdown

```yaml
server:
  shutdown: graceful
spring:
  lifecycle:
    timeout-per-shutdown-phase: 30s
```

readiness가 먼저 DOWN → 트래픽 차단 → in-flight 요청 완료 → 종료.
