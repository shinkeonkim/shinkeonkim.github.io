## 정의

**SQL** (Structured Query Language) 은 관계형 데이터베이스 (RDBMS) 와 대화하기 위한 선언형 (declarative) 질의 언어. 1970 년 E.F. Codd 의 관계 모델 논문에서 출발, 1986 년 ANSI 표준화 (SQL-86), 이후 SQL:2023 까지 개정.

"무엇을 (what)" 만 적고 "어떻게 (how)" 는 옵티마이저가 결정한다. 같은 결과를 만드는 실행 경로는 수천 가지일 수 있고, RDBMS 가 통계 기반으로 가장 빠른 길을 고른다.

## 4 개의 하위 언어

표준 SQL 은 목적에 따라 4 개 카테고리로 나뉜다.

| 카테고리 | 풀네임 | 키워드 | 용도 |
|:---|:---|:---|:---|
| **DDL** | Data Definition Language | `CREATE`, `ALTER`, `DROP`, `TRUNCATE` | 스키마 정의 |
| **DML** | Data Manipulation Language | `SELECT`, `INSERT`, `UPDATE`, `DELETE` | 데이터 조작 |
| **DCL** | Data Control Language | `GRANT`, `REVOKE` | 권한 제어 |
| **TCL** | Transaction Control Language | `BEGIN`, `COMMIT`, `ROLLBACK`, `SAVEPOINT` | 트랜잭션 제어 |

각 항목은 별도 wiki 로 깊이 다룬다, [[SQL DDL]] / [[SQL DML]] / [[SQL DCL TCL]].

> 일부 문헌은 `SELECT` 를 DQL (Data Query Language) 로 분리하기도 한다. ISO 표준은 DML 에 포함시킨다.

## 쿼리 실행의 논리적 순서

작성 순서와 실행 순서는 다르다. `SELECT` 를 먼저 쓰지만, 옵티마이저는 `FROM` 부터 평가한다.

```anim:sql-query-order
{}
```

| 순서 | 절 | 의미 |
|:---:|:---|:---|
| 1 | `FROM` / `JOIN` | 소스 테이블 결합 |
| 2 | `WHERE` | 행 필터 (그룹화 전) |
| 3 | `GROUP BY` | 그룹 분할 |
| 4 | `HAVING` | 그룹 필터 |
| 5 | `SELECT` | 컬럼 선택, 표현식 평가 |
| 6 | `DISTINCT` | 중복 제거 |
| 7 | `ORDER BY` | 정렬 |
| 8 | `LIMIT` / `OFFSET` | 페이징 |

> [!IMPORTANT]
> `WHERE` 는 집계 결과를 참조할 수 없다 (`SELECT` 보다 먼저 실행되니까). 집계 조건은 `HAVING` 에 쓴다.

## pandas 와의 1:1 대응

[[Pandas]] 의 DataFrame 연산은 SQL 과 거의 대칭이다. 같은 개념의 두 표현.

| SQL | pandas | 비고 |
|:---|:---|:---|
| `SELECT col1, col2 FROM t` | `df[['col1', 'col2']]` | 컬럼 선택 |
| `WHERE col > 10` | `df[df['col'] > 10]` | [[Pandas Boolean Indexing]] |
| `ORDER BY col DESC` | `df.sort_values('col', ascending=False)` | 정렬 |
| `LIMIT 10` | `df.head(10)` | 상위 N |
| `DISTINCT col` | `df['col'].unique()` / `drop_duplicates` | 중복 제거 |
| `GROUP BY` + `AVG(x)` | `df.groupby(...).mean()` | [[Pandas groupby]] |
| `INNER JOIN` | `pd.merge(a, b, how='inner')` | [[Pandas merge]] |
| `LEFT JOIN` | `pd.merge(a, b, how='left')` | [[Pandas merge]] |
| `UNION ALL` | `pd.concat([a, b])` | [[Pandas concat]] |
| `CASE WHEN` | `np.where` / `pd.cut` | 조건 분기 |
| 윈도 `ROW_NUMBER()` | `df.groupby(...).cumcount()` | 그룹 내 순번 |
| `INSERT INTO` | `df.to_sql(...)` | [[Pandas read_sql]] |

DB 와 직접 주고받는 통로는 [[Pandas read_sql]] 참고.

## 표준 SQL vs dialect

ANSI/ISO 표준은 골격, 실제 구현은 dialect 마다 차이가 크다.

| 영역 | 표준 (SQL:2023) | PostgreSQL | MySQL |
|:---|:---|:---|:---|
| 자동 증가 PK | `IDENTITY` | `SERIAL`, `GENERATED ... AS IDENTITY` | `AUTO_INCREMENT` |
| 문자열 결합 | `||` | `||`, `CONCAT()` | `CONCAT()` (`||` 는 기본 OR) |
| 상위 N | `FETCH FIRST n ROWS ONLY` | `LIMIT n` (+ `FETCH`) | `LIMIT n` |
| Upsert | `MERGE` | `INSERT ... ON CONFLICT` | `INSERT ... ON DUPLICATE KEY UPDATE` |
| 부울 | `BOOLEAN` | `BOOLEAN` (네이티브) | `BOOLEAN` ≡ `TINYINT(1)` |
| JSON | `JSON` | `JSON`, `JSONB` (인덱싱 가능) | `JSON` (인덱싱은 generated column) |
| 정규식 | `LIKE_REGEX` (거의 미구현) | `~`, `~*`, `regexp_*` | `REGEXP`, `RLIKE` |

상세는 [[PostgreSQL]], [[MySQL]] 참고.

## 데이터 타입 그룹

연산은 타입에 따라 가능 여부와 결과가 달라진다.

| 그룹 | 예 | 주요 연산 |
|:---|:---|:---|
| **숫자** | `INTEGER`, `BIGINT`, `NUMERIC`, `DOUBLE` | 산술, 집계, `ROUND/CEIL/FLOOR` |
| **문자열** | `VARCHAR`, `TEXT`, `CHAR` | `LIKE`, 정규식, `CONCAT`, `SUBSTRING` |
| **날짜/시간** | `DATE`, `TIMESTAMP`, `TIMESTAMPTZ`, `INTERVAL` | 차이, 추출, 트런케이션, `INTERVAL` 산술 |
| **부울** | `BOOLEAN` | `AND`, `OR`, `NOT` |
| **이진** | `BYTEA` (PG), `BLOB` (MySQL) | 길이, 해시, 인코딩 |
| **JSON** | `JSON`, `JSONB` (PG) | 경로 추출, containment, 인덱스 |
| **배열** (PG) | `INTEGER[]`, `TEXT[]` | `ANY/ALL`, 슬라이싱, `unnest` |
| **공간** | `GEOMETRY`, `GEOGRAPHY` (PostGIS) | 거리, 포함, 교차 |

## NULL 의 3-값 논리

SQL 의 가장 까다로운 부분.

```sql
SELECT NULL = NULL;        -- NULL (FALSE 가 아님)
SELECT NULL <> NULL;       -- NULL
SELECT NULL IS NULL;       -- TRUE
SELECT 1 + NULL;           -- NULL
SELECT COUNT(*) FROM t;    -- NULL 포함
SELECT COUNT(col) FROM t;  -- NULL 제외
```

- 비교 결과가 `TRUE` / `FALSE` / `NULL` 3 가지
- `WHERE` 절은 `TRUE` 인 행만 통과 (`NULL` 도 제외)
- 집계 함수는 `NULL` 을 자동 무시 (단 `COUNT(*)` 는 행 자체를 셈)
- `NULL` 비교는 `IS NULL` / `IS NOT NULL` / `IS DISTINCT FROM`
