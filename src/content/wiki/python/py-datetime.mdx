## 정의

`datetime` 모듈은 날짜·시각·기간 처리를 제공한다. 핵심 클래스:

- `date`: 연-월-일
- `time`: 시-분-초-마이크로초 (날짜 정보 없음)
- `datetime`: date + time
- `timedelta`: 기간 (날짜 간 차이)
- `timezone`: UTC 오프셋 기반 타임존
- `zoneinfo.ZoneInfo` (3.9+): IANA 타임존 DB ("Asia/Seoul" 등)

## naive vs aware

가장 중요한 구분.

- **naive datetime**: 타임존 정보 없음. `tzinfo is None`
- **aware datetime**: 타임존 정보 있음. UTC 환산 가능

```python
from datetime import datetime, timezone

naive = datetime(2026, 6, 20, 12, 0)
print(naive.tzinfo)        # None

aware = datetime(2026, 6, 20, 12, 0, tzinfo=timezone.utc)
print(aware.tzinfo)        # UTC
```

**서버/저장은 aware UTC가 표준**. naive는 로컬 표현이나 입력 파싱 직후 등 한정적으로만.

```python
# UTC now (aware) - 권장
now = datetime.now(timezone.utc)

# 로컬 now (naive) - 비권장 (TZ 모호)
now = datetime.now()
```

## 출력: isoformat과 strftime

```python
dt = datetime(2026, 6, 20, 14, 30, tzinfo=timezone.utc)

print(dt.isoformat())              # '2026-06-20T14:30:00+00:00'
print(dt.strftime("%Y-%m-%d %H:%M %z"))   # '2026-06-20 14:30 +0000'
print(dt.strftime("%A, %B %d"))    # 'Saturday, June 20'
```

ISO 8601 형식이 직렬화 표준. `isoformat()`을 우선.

### 일광절약시간(DST)

```python
from datetime import datetime, timedelta
from zoneinfo import ZoneInfo

ny = ZoneInfo("America/New_York")
spring = datetime(2026, 3, 8, 1, 30, tzinfo=ny)  # 02:00에 03:00으로 점프
later = spring + timedelta(hours=2)
print(later)    # 2026-03-08 04:30 EDT
```

`zoneinfo`가 DST 전환을 자동 처리. timezone 오프셋이 시기별로 다르다.

## astimezone vs replace

```python
dt = datetime(2026, 6, 20, 14, 0, tzinfo=ZoneInfo("UTC"))

# astimezone: 같은 순간을 다른 TZ로 표현
dt.astimezone(ZoneInfo("Asia/Seoul"))     # 2026-06-20 23:00 KST

# replace: TZ만 바꿈 (실제 순간이 달라짐!) - 위험
dt.replace(tzinfo=ZoneInfo("Asia/Seoul")) # 2026-06-20 14:00 KST (다른 순간!)
```

`replace`는 naive에 TZ를 입힐 때만 사용. 이미 aware인 dt를 다른 TZ로 보려면 항상 `astimezone`.

## UNIX timestamp 변환

```python
from datetime import datetime, timezone

# datetime → timestamp
ts = datetime.now(timezone.utc).timestamp()
print(ts)    # 1750395600.123456 (float)

# timestamp → datetime
dt = datetime.fromtimestamp(ts, tz=timezone.utc)
print(dt)

# naive timestamp는 로컬 TZ로 해석되어 혼란 위험
naive = datetime.fromtimestamp(ts)   # 로컬 TZ 해석
```

`timestamp()`는 항상 UTC epoch 초. 안전.

## 비교와 정렬

`datetime`은 비교 가능. 단 naive와 aware 비교는 `TypeError`.

```python
now = datetime.now(timezone.utc)
later = now + timedelta(hours=1)
print(now < later)    # True

# WRONG: 혼합
naive = datetime.now()
aware = datetime.now(timezone.utc)
naive < aware    # TypeError
```

## 자주 보는 함정

### 1. `datetime.utcnow()` 비권장

```python
# 비권장 (naive 반환)
datetime.utcnow()      # tzinfo=None인데 사실 UTC 시각... 함정!

# 권장
datetime.now(timezone.utc)    # aware UTC
```

3.12+ 부터 `utcnow()`는 deprecated.

### 2. `datetime.fromtimestamp()` 기본은 로컬 TZ

```python
datetime.fromtimestamp(ts)               # 로컬 TZ로 해석 → naive
datetime.fromtimestamp(ts, tz=timezone.utc)   # UTC aware (권장)
```

### 3. ISO 파싱 한계

`fromisoformat`(3.7+)은 엄격한 ISO 8601만. 3.11+에서 더 관대해졌지만 여전히 `Z`(UTC) 제대로 처리하려면 3.11+ 필요.

```python
# 3.10 이하에선 'Z' 안 됨
datetime.fromisoformat("2026-06-20T14:30:00Z")   # ValueError (3.10-)
datetime.fromisoformat("2026-06-20T14:30:00+00:00")   # OK
```

전 버전 호환은 `dateutil.parser.parse` 또는 `pendulum`.

### 4. month/year 더하기

`timedelta`엔 `months`나 `years`가 없다 (월/년은 길이 가변).

```python
# 한 달 후?
from dateutil.relativedelta import relativedelta
dt + relativedelta(months=1)

# 또는 직접 계산
def add_months(dt, months):
    year = dt.year + (dt.month - 1 + months) // 12
    month = (dt.month - 1 + months) % 12 + 1
    day = min(dt.day, last_day_of_month(year, month))
    return dt.replace(year=year, month=month, day=day)
```

## time vs datetime

| 모듈 | 용도 |
|------|------|
| `time` | 낮은 수준, UNIX timestamp, monotonic clock |
| `datetime` | 사람이 읽는 날짜·시각 |
| `calendar` | 달력 (월 길이, 윤년) |

```python
import time
time.time()              # UNIX timestamp (float)
time.monotonic()         # 단조 증가 (시간 차이 측정용, 절대 시각 아님)
time.perf_counter()      # 고해상도 (성능 측정)
time.sleep(0.5)
```

벤치마크엔 `perf_counter`. 사용자 시각 표시엔 `datetime`.
