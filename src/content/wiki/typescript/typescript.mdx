## 정의

**TypeScript** 는 Microsoft 가 2012년 발표한 **JavaScript 의 상위 집합** 프로그래밍 언어입니다. 정적 타입 시스템, 인터페이스, 제네릭, enum, 데코레이터, 모듈 시스템 등을 추가하며, 컴파일 시 순수 JavaScript 로 변환됩니다.

**핵심 원리**: TypeScript 는 **JavaScript + 타입 어노테이션**. 런타임에는 타입이 지워집니다 (`type erasure`). 타입은 오직 컴파일 시 정적 검사와 IDE 지원 목적입니다.

## 왜 TypeScript 인가

### 1. 정적 타입 검사

```typescript
function greet(name: string): string {
  return `Hello, ${name}`;
}

greet(42);   // 컴파일 오류
```

버그를 컴파일 시점에 잡습니다. 대규모 코드베이스에서 리팩토링, 함수 시그니처 변경 안전성이 크게 향상.

### 2. IDE 지원

- 자동완성 (엄밀한 시그니처 기반)
- Rename symbol (다중 파일 안전)
- Go to definition
- Find all references
- Inline hint

### 3. 명세로서의 타입

함수 시그니처가 문서 역할. `function process(items: Item[], options?: Options): Promise<Result[]>` 만 봐도 사용법 이해.

### 4. 마이그레이션 유연성

- **점진 도입**: `.js` 를 `.ts` 로 하나씩 변환
- **`any` 로 임시 회피** 후 점진 좁히기
- **JSDoc** 로 순수 JS 에도 부분 타입 지원

### 5. 방대한 생태계

- **[DefinitelyTyped](https://github.com/DefinitelyTyped/DefinitelyTyped)**: 8,000+ npm 패키지의 타입 정의
- 주요 프레임워크 (React, Vue, Angular, Next.js, NestJS, Deno, Bun) 가 TS first

## Hello World

```bash
npm install -D typescript
npx tsc --init          # tsconfig.json 생성
```

```typescript
// hello.ts
const greeting: string = "Hello, TypeScript";
console.log(greeting);

interface User {
  id: number;
  name: string;
}

function greet(user: User): string {
  return `Hi, ${user.name}`;
}

greet({id: 1, name: "kim"});
```

```bash
npx tsc hello.ts        # -> hello.js
node hello.js
```

## 컴파일 파이프라인

```
[.ts / .tsx 소스]
     ↓ tsc (또는 esbuild, swc, bun)
[.js / .jsx]
     ↓ 번들러 (Webpack, Vite, esbuild, Rollup)
[번들.js]
```

- **`tsc`**: 공식 컴파일러. 타입 검사 + JS 생성.
- **`esbuild` / `swc`**: 매우 빠른 컴파일러. 타입 검사 없이 JS 변환만 (검사는 `tsc --noEmit` 별도).
- **`bun`**: 자체 컴파일러 + 런타임.
- **`ts-node`, `tsx`**: 개발 시 즉시 실행.

**관용**: 프로덕션은 `tsc` (검사) + `esbuild`/`vite` (번들) 조합. `tsc` 는 검사만 하고, 실제 변환은 빠른 도구.

## 타입 시스템 개요

TypeScript 의 타입 시스템은 **structural typing** ("duck typing" 정적 버전).

```typescript
interface Named { name: string }
interface Aged { age: number }

class Person {
  name: string = "";
  age: number = 0;
}

const p: Named = new Person();   // OK, Person 이 name 을 가짐
const a: Aged = new Person();    // OK, Person 이 age 를 가짐
```

**Java/C# 의 nominal typing** (이름 기반) 과 다름. 이름이 아니라 shape (형태) 로 판단.

## 주요 타입 카테고리

- **[[ts-primitive-types|Primitive]]**: string, number, boolean, bigint, symbol, null, undefined, void, never, unknown, any
- **Object**: `{name: string}`, `Record<K, V>`
- **Array**: `number[]`, `Array<number>`
- **Tuple**: `[string, number]`
- **Function**: `(a: number, b: number) => number`
- **Union / Intersection**: `A | B`, `A & B` (참고: [[ts-union-intersection|Union/Intersection]])
- **[[ts-generics|Generic]]**: `<T>`, `<T extends string>`
- **Literal**: `"hello"`, `42`, `true`
- **Template Literal**: `` `hello_${string}` `` (참고: [[ts-template-literal-types|Template Literal Types]])
- **Enum** (참고: [[ts-enums|Enums]])

## 3가지 특별 타입

### `any`

**모든 타입 무시**. 타입 안전성을 끔.

```typescript
let x: any = 5;
x = "hello";
x.foo.bar();   // 컴파일 통과, 런타임 오류
```

`any` 는 급한 마이그레이션 시 임시 도구. 프로덕션 코드에서 남용 금지. `strict` 모드에서 amberflex enabled 하는 것이 관용.

### `unknown`

**타입을 모른다**. 하지만 `any` 와 달리 **narrowing 없이 사용 불가**:

```typescript
let x: unknown = 5;
x.toUpperCase();          // 오류: unknown 에는 접근 불가

if (typeof x === "string") {
  x.toUpperCase();        // OK: narrowing 후
}
```

외부에서 오는 값 (JSON.parse, API 응답) 은 `any` 대신 `unknown` 을 쓰고 검증 후 좁히는 것이 안전.

### `never`

**절대 발생하지 않는 값**. 오류 던지는 함수, exhaustive check 에 쓰임.

```typescript
function assertNever(x: never): never {
  throw new Error(`Unexpected: ${x}`);
}

type Shape = {kind: "circle", r: number} | {kind: "square", s: number};

function area(s: Shape) {
  switch (s.kind) {
    case "circle": return Math.PI * s.r ** 2;
    case "square": return s.s ** 2;
    default: return assertNever(s);   // 새 case 추가 시 컴파일 오류
  }
}
```

## tsconfig.json

TypeScript 프로젝트 설정. 핵심 옵션:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "esnext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,               // esbuild 가 emit 하는 경우
    "jsx": "react-jsx",
    "lib": ["ES2022", "DOM"]
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

자세한 것은 [[ts-strict-mode|Strict Mode & tsconfig]] 참조.

## 주요 문법 요약

### 변수 타입

```typescript
let name: string = "kim";
let age: number = 30;
let isAdmin: boolean = false;
let ids: number[] = [1, 2, 3];
let user: {name: string, age: number} = {name: "kim", age: 30};
```

타입 추론 (inference) 이 강력하므로 대개 명시 불필요:

```typescript
const name = "kim";     // 자동으로 string 리터럴 타입
let count = 0;          // let 은 number
```

### 함수

```typescript
function add(a: number, b: number): number {
  return a + b;
}

const multiply = (a: number, b: number): number => a * b;

// 매개변수 기본값
function greet(name: string = "world"): string {
  return `Hello, ${name}`;
}

// Optional
function log(msg: string, level?: string): void {
  ...
}

// Rest
function sum(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0);
}
```

### 클래스

```typescript
class Animal {
  protected name: string;
  private _id: number;

  constructor(name: string) {
    this.name = name;
    this._id = Math.random();
  }

  greet(): void {
    console.log(`Hi, ${this.name}`);
  }
}

class Dog extends Animal {
  bark(): void {
    console.log("Woof!");
  }
}
```

Constructor parameter properties:

```typescript
class User {
  constructor(
    public name: string,
    private age: number,
    readonly id: number,
  ) {}
}
// name, age, id 자동으로 인스턴스 필드
```

### 인터페이스 vs Type alias

```typescript
interface User {
  id: number;
  name: string;
}

type User = {
  id: number;
  name: string;
};
```

거의 상호 교환. 차이점은 [[ts-interfaces|Interfaces]] 와 [[ts-type-aliases|Type Aliases]] 참조.

## TypeScript 언어 발전 (최근)

- **5.0 (2023)**: 표준 Decorators (TC39 Stage 3), `const` type parameter, all-in-one moduleResolution
- **5.1 (2023)**: `@returns` return type
- **5.2 (2023)**: `using` (explicit resource management)
- **5.3 (2023)**: Import attributes
- **5.4 (2024)**: `NoInfer<T>` 타입
- **5.5 (2024)**: Inferred type predicates, `Set` methods
- **5.6 (2024)**: Disallowed nullish and truthy checks
- **5.7 (2024)**: Path Rewriting, checks for never-initialized variables
- **6.0 (2025+ 예정)**

Release note 는 [TypeScript 사이트](https://www.typescriptlang.org/docs/handbook/release-notes/) 참조.

## TypeScript vs 다른 언어

| 기준 | TS | Flow | Dart | Kotlin |
|:---|:---|:---|:---|:---|
| **관계** | JS 상위 집합 | JS 상위 집합 | 별개 언어 | JVM 언어 |
| **채택** | 지배적 | 감소 | Flutter 특화 | Android/서버 |
| **타입 시스템** | 강력, 실용적 | 유사 | 유사 | 유사 |
| **트랜스파일** | JS | JS | JS/native | JVM/JS/native |

**결론**: 2020년대 웹/Node.js 프로젝트의 사실상 표준.
