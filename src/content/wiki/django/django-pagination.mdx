## 정의

Django는 빌트인 `Paginator` 클래스와 ListView의 자동 페이지네이션을 제공한다. DRF는 PageNumber / LimitOffset / Cursor 3종. 페이지네이션은 큰 결과셋의 메모리·응답 시간 보호의 기본.

## 빌트인 Paginator

```python
from django.core.paginator import Paginator, EmptyPage, PageNotAnInteger

def post_list(request):
    posts = Post.objects.all().order_by("-created_at")
    paginator = Paginator(posts, 20)    # 페이지당 20개

    page_number = request.GET.get("page", 1)
    try:
        page_obj = paginator.page(page_number)
    except PageNotAnInteger:
        page_obj = paginator.page(1)
    except EmptyPage:
        page_obj = paginator.page(paginator.num_pages)

    return render(request, "posts.html", {"page_obj": page_obj})
```

또는 더 간단히 `paginator.get_page(page_number)`:

```python
page_obj = paginator.get_page(page_number)    # 자동 처리
```

## ListView 자동

```python
class PostListView(ListView):
    model = Post
    paginate_by = 20
    template_name = "posts.html"
    context_object_name = "posts"
```

자동으로 `page_obj`, `paginator`, `is_paginated` context 제공. 쿼리스트링 `?page=2`.

## DRF Pagination

### PageNumberPagination (기본)

```python
# settings.py
REST_FRAMEWORK = {
    "DEFAULT_PAGINATION_CLASS": "rest_framework.pagination.PageNumberPagination",
    "PAGE_SIZE": 20,
}
```

요청: `?page=2`, 응답:

```json
{
    "count": 100,
    "next": "https://.../?page=3",
    "previous": "https://.../?page=1",
    "results": [...]
}
```

### 커스텀 (page_size_query_param)

```python
class StandardPagination(PageNumberPagination):
    page_size = 20
    page_size_query_param = "page_size"    # ?page_size=50
    max_page_size = 100

# ViewSet에서
class PostViewSet(ModelViewSet):
    pagination_class = StandardPagination
```

### LimitOffsetPagination

```python
REST_FRAMEWORK = {
    "DEFAULT_PAGINATION_CLASS": "rest_framework.pagination.LimitOffsetPagination",
}
```

요청: `?limit=20&offset=40`, 응답:

```json
{
    "count": 100,
    "next": "?limit=20&offset=60",
    "previous": "?limit=20&offset=20",
    "results": [...]
}
```

OFFSET이 크면 SQL 느려짐. 깊은 페이지에 부적합.

### CursorPagination

```python
class PostCursorPagination(CursorPagination):
    ordering = "-created_at"    # 필수 (unique 정렬 권장)
    page_size = 20
```

응답:
```json
{
    "next": "https://.../?cursor=cD0yMDI2LTA2LTIw...",
    "previous": null,
    "results": [...]
}
```

장점:
- 안정적 (행 추가/삭제 영향 X)
- O(log n) 성능 (인덱스 활용)
- 깊은 페이지도 빠름

단점: 임의 페이지 점프 X (순차만). 무한 스크롤·실시간 피드에 최적.

## 함정

### 1. order_by 없이 페이지네이션

```python
Paginator(Post.objects.all(), 20)    # 정렬 없음 → 순서 보장 X
```

기본 정렬이 없으면 페이지마다 같은 행이 중복 표시되거나 누락. 항상 정렬.

### 2. queryset이 list로 변환됨

```python
posts = list(Post.objects.all())    # 모두 로드
paginator = Paginator(posts, 20)    # 비효율
```

QuerySet 그대로 전달 → lazy.

### 3. CursorPagination + 비유일 정렬

```python
ordering = "created_at"    # 같은 시각 행 여러 개면 cursor 깨짐
ordering = ("-created_at", "id")    # 유일성 보장
```

### 4. SEO와 페이지네이션

큰 페이지 수는 검색 엔진이 다 크롤링 못 함. canonical/prev/next 헤더 또는 사이트맵.

### 5. DRF count 정확성

분산 환경에서 COUNT 정합성 보장 X. 실시간 데이터는 cursor 또는 estimate.
