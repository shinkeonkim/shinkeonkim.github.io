## 정의

**`java.util.Queue<E>`** 는 **FIFO (First-In, First-Out)** 처리 순서를 가진 [[Collection]] 인터페이스. 한쪽 끝 (tail) 에 추가하고 반대쪽 (head) 에서 제거하는 자료구조.

`PriorityQueue` 처럼 다른 순서 (heap) 인 구현체도 있지만, 인터페이스 자체는 "원소를 꺼내는 순서를 정의" 정도의 추상.

## 두 그룹의 메서드

각 동작마다 **예외를 던지는 버전** 과 **특수값을 반환하는 버전** 이 있다.

| 동작 | 예외 throw | 특수값 |
|:---|:---|:---|
| insert | `add(e)` | `offer(e)` |
| remove | `remove()` | `poll()` |
| examine | `element()` | `peek()` |

- **`add` / `remove` / `element`**: 큐가 가득 차거나 비어 있을 때 예외
- **`offer` / `poll` / `peek`**: 같은 상황에서 `false`, `null` 반환

대부분의 코드에서는 **null/false 반환 버전이 안전** 하다. `add` 가 가득 찰 때 던지는 예외를 제대로 처리하는 코드는 흔치 않다.

## 구현체

| 구현 | 백킹 | 특징 |
|:---|:---|:---|
| **[[LinkedList]]** | 양방향 연결 리스트 | 가장 단순한 unbounded queue |
| **[[ArrayDeque]]** | 원형 배열 | 빠른 deque, 권장 |
| **[[PriorityQueue]]** | 힙 | 우선순위 순으로 꺼냄 |
| **[[ConcurrentLinkedQueue]]** | linked list (lock-free) | thread-safe, unbounded |
| **[[BlockingQueue]] 구현체** | 다양 | 가득/비어 있으면 block |

## Deque 와의 관계

`Queue` 의 확장 [[Deque]] (Double-Ended Queue) 는 양 끝 모두에서 add/remove 가능. ArrayDeque, LinkedList 등이 Deque 를 구현.

`Queue<E> q = new ArrayDeque<>()` 처럼 인터페이스 타입을 `Queue` 로 잡으면 FIFO 만의 약속을 사용한다.
