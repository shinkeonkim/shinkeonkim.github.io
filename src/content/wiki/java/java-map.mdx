## 정의

**`java.util.Map<K,V>`** 는 **key 와 value 의 쌍으로 데이터를 저장** 하는 자료구조의 인터페이스. **하나의 key 에는 하나의 value 만 매핑**, 중복 key 는 허용하지 않는다.

`Map` 은 [[Collection]] 을 extends 하지 **않는다** (key-value 쌍 의미가 단일 원소와 다르므로). 그래도 `keySet()`, `values()`, `entrySet()` 을 통해 `Collection` 인터페이스의 view 를 제공.

## 핵심 메서드

| 메서드 | 의미 |
|:---|:---|
| `put(K, V)` | 매핑 추가 또는 갱신, 이전 값 반환 |
| `get(Object)` | key 로 value 조회, 없으면 null |
| `getOrDefault(Object, V)` | 없으면 기본값 |
| `containsKey(Object)` | key 존재 여부 |
| `containsValue(Object)` | value 존재 여부 (보통 O(n)) |
| `remove(Object)` | key 제거 |
| `size()`, `isEmpty()` | 크기 |
| `keySet()`, `values()`, `entrySet()` | view 들 |

### Java 8+ 의 기본 메서드

```java
map.putIfAbsent(key, value);
map.computeIfAbsent(key, k -> expensive(k));
map.computeIfPresent(key, (k, v) -> v + 1);
map.compute(key, (k, v) -> v == null ? 1 : v + 1);
map.merge(key, 1, Integer::sum);
map.replace(key, oldVal, newVal);
```

## 구현체 한눈에

| 구현 | 순서 | thread-safe | null 허용 | 백킹 |
|:---|:---|:---:|:---|:---|
| **[[HashMap]]** | 없음 | ✗ | key 1, value 다수 | 해시 테이블 |
| **[[LinkedHashMap]]** | 삽입 순서 또는 접근 순서 | ✗ | key 1, value 다수 | 해시 + linked list |
| **[[TreeMap]]** | 정렬 (Comparable / Comparator) | ✗ | value 다수, key 는 null 비허용 | Red-Black Tree |
| **`Hashtable`** (legacy) | 없음 | ✓ (메서드) | 둘 다 비허용 | 해시 |
| **[[ConcurrentHashMap]]** | 없음 | ✓ (bin-level) | 둘 다 비허용 | 해시 + CAS + sync |
| **[[Spring] ConcurrentReferenceHashMap]] | 없음 | ✓ | 비허용 | 해시 + Weak/Soft ref |
| **`EnumMap`** | enum 정의 순서 | ✗ | value 다수 | 배열 |
| **`IdentityHashMap`** | 없음 | ✗ | 다수 | `==` 비교 |
| **`WeakHashMap`** | 없음 | ✗ | 다수 | key 약참조 |

## 불변 Map (Java 9+)

```java
Map<String, Integer> ages = Map.of("Alice", 30, "Bob", 25);
Map<String, Integer> more = Map.ofEntries(
    Map.entry("a", 1),
    Map.entry("b", 2)
);
```

10개 이상이거나 일관성 있게 표현하려면 `ofEntries`.

## entrySet 순회가 가장 효율적

```java
// ❌ 비효율 (key 로 다시 get)
for (String k : map.keySet()) {
    System.out.println(k + "=" + map.get(k));
}

// ✓ entrySet 직접 순회 (해시 lookup 1번)
for (Map.Entry<String, Integer> e : map.entrySet()) {
    System.out.println(e.getKey() + "=" + e.getValue());
}

// ✓ Java 8+ forEach
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

## 동기화 옵션

```java
Map<K, V> sync = Collections.synchronizedMap(new HashMap<>());
Map<K, V> concurrent = new ConcurrentHashMap<>();  // 권장
```

[[ConcurrentHashMap]] 이 거의 항상 더 빠르다.
