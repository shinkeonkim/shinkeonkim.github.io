## 정의

JS 의 제어 흐름 구문. 분기, 반복, 종료 / 계속.

## if / else

```javascript
if (cond) { ... }
else if (cond2) { ... }
else { ... }

// 단일 표현식이면 ternary 권장
const x = cond ? a : b;
```

## switch

```javascript
switch (x) {
    case 1:
        // fall-through 주의
        break;
    case 2:
    case 3:
        // 2 or 3
        break;
    default:
        // ...
}
```

`break` 없으면 다음 case 로 fall-through. ES2023+ `Map` lookup 이 더 명시적.

```javascript
// switch 대안
const action = {
    create: () => create(),
    update: () => update(),
    delete: () => del(),
}[type];
action?.();
```

## for

```javascript
for (let i = 0; i < n; i++) { ... }

// 무한 + break
for (;;) {
    if (cond) break;
}
```

## for...of (iterable)

```javascript
for (const x of arr) { ... }
for (const [k, v] of map) { ... }
for (const ch of 'hello') { ... }
```

`break`, `continue` 지원. `index` 가 필요하면 `arr.entries()` 또는 일반 for.

```javascript
for (const [i, v] of arr.entries()) { ... }
```

## for...in (객체 키)

```javascript
for (const key in obj) { ... }
```

> [!CAUTION]
> **`for...in` 은 prototype chain 의 키도 포함**. 배열에 쓰지 말 것 (의도와 다른 순서). 객체에는 `Object.keys/entries` 가 더 안전.

## while / do-while

```javascript
while (cond) { ... }

do {
    // 최소 1 회 실행
} while (cond);
```

## break / continue

```javascript
for (...) {
    if (skip) continue;
    if (stop) break;
}

// label 로 nested loop 제어
outer: for (...) {
    inner: for (...) {
        if (cond) break outer;
        if (cond2) continue outer;
    }
}
```

label 은 가독성 떨어져 보통 회피, 함수로 추출이 더 깔끔.

## ternary

```javascript
const x = cond ? a : b;
const tag = age >= 18 ? 'adult' : age >= 13 ? 'teen' : 'child';  // chained
```

## logical operators 의 short-circuit

```javascript
a && b           // a 가 truthy 면 b 평가
a || b           // a 가 truthy 면 a
a ?? b           // a 가 nullish 면 b
```

### conditional execution

```javascript
isLoggedIn && showProfile();
config.debug && console.log(...);

// 옛 if 패턴
if (isLoggedIn) showProfile();
```

`if` 가 더 명시적, lint 도구 (no-unused-expressions) 가 short-circuit 패턴 경고할 수 있음.

## return early (가독성)

```javascript
// ❌ 중첩
function process(user) {
    if (user) {
        if (user.active) {
            if (user.verified) {
                return doWork(user);
            }
        }
    }
    return null;
}

// ✓ early return
function process(user) {
    if (!user) return null;
    if (!user.active) return null;
    if (!user.verified) return null;
    return doWork(user);
}
```

## 자주 만나는 패턴

### Array methods 가 for 보다 깔끔

```javascript
// for
const result = [];
for (const x of arr) {
    if (x > 0) result.push(x * 2);
}

// 함수형
const result = arr.filter(x => x > 0).map(x => x * 2);
```

### 조건부 객체 빌딩

```javascript
const obj = {
    a: 1,
    ...(cond && { b: 2 }),
    ...(other && { c: 3 }),
};
```

### 모드별 분기

```javascript
const handlers = {
    pending: handlePending,
    success: handleSuccess,
    error: handleError,
};
handlers[state]?.(payload);
```

## 함정

### 1. ==/!= 의 강제 변환

```javascript
if (x == 0) { ... }      // '' 0 false null 등 모두 match
if (x === 0) { ... }     // 정확
```

### 2. switch 의 fallthrough

```javascript
switch (x) {
    case 1:
        doOne();
        // break 빠뜨림 → 다음 case 실행
    case 2:
        doTwo();
}
```

의도라면 주석 `// fallthrough` 명시.

### 3. for-in 의 prototype 키

```javascript
Array.prototype.custom = function() {};
for (const k in [1, 2, 3]) console.log(k);
// '0', '1', '2', 'custom'   ⚠️
```

`for...of` 또는 `for (let i = 0; ...)` 권장.

### 4. 빈 catch

```javascript
try { ... } catch {}    // ⚠️ 에러 무시 위험
```
