## 정의

- **`Series.unique()`** : 고유값을 **등장 순서대로** numpy array 반환
- **`Series.nunique()`** : 고유값의 **개수** 반환 (NaN 제외)
- **`DataFrame.nunique()`** : 각 컬럼/행의 고유값 개수

SQL 의 `SELECT DISTINCT` / `COUNT(DISTINCT)` 와 동일.

## DataFrame.nunique

```python
df.nunique()                # 각 컬럼의 고유값 개수
df.nunique(axis=1)          # 각 행의 고유값 개수
df.nunique(dropna=False)
```

## drop_duplicates 와의 관계

- `unique` : Series 의 고유값 자체 (정렬되지 않은 원순서)
- `drop_duplicates` : DataFrame/Series 의 **행 단위** 중복 제거
- `nunique` : 고유값 **개수만**

자세한 행 단위 중복 제거는 [[Pandas drop_duplicates]] 참고.

## 정렬된 고유값

`unique()` 는 등장 순서. 정렬이 필요하면:

```python
sorted(s.unique())              # Python sorted
np.sort(s.unique())             # numpy sort
s.unique()                       # 원순서 (등장 순)
```

## 활용

### groupby + nunique (그룹별 고유값 개수)

```python
df.groupby('city')['user_id'].nunique()      # 도시별 고유 사용자 수
df.groupby('city').nunique()                  # 도시별 모든 컬럼의 nunique
```

SQL: `SELECT city, COUNT(DISTINCT user_id) FROM ... GROUP BY city`

### unique 의 결과는 numpy array

```python
arr = s.unique()              # numpy ndarray (dtype 보존)
ser = pd.Series(s.unique())   # 다시 Series 로
```

## 함정

### 1. unique 의 정렬 안 됨

`unique` 가 정렬된 결과를 반환할 거라 가정하면 안 된다.

### 2. NaN 처리

```python
s.unique()                    # NaN 도 포함 (numpy nan)
s.nunique()                   # NaN 제외 (default)
s.nunique(dropna=False)       # NaN 포함
```

### 3. 대용량 메모리

```python
df.nunique()                  # 모든 컬럼에 대해 unique 계산, 메모리 多
df['col'].nunique()           # 한 컬럼만 (권장)
```
