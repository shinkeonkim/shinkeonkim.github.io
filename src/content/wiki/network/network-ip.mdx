## 정의

**IP (Internet Protocol) 주소** 는 인터넷에서 노드를 식별하는 논리 주소. IPv4 (32bit) 와 IPv6 (128bit) 두 버전이 병존.

## IPv4

**4 개 8bit 옥텟**: `192.168.1.1`. 총 2³² ≈ 43억 개, 이미 고갈.

### 표준 주소 범위

| 클래스 | 범위 | 용도 |
|:---|:---|:---|
| 사설 A | 10.0.0.0/8 | 대규모 사내망 |
| 사설 B | 172.16.0.0/12 | |
| 사설 C | 192.168.0.0/16 | 홈 라우터 기본 |
| 루프백 | 127.0.0.0/8 | localhost |
| 링크 로컬 | 169.254.0.0/16 | APIPA |
| 멀티캐스트 | 224.0.0.0/4 | |

### CIDR 표기

`192.168.1.0/24` = 앞 24 bit 가 네트워크, 뒤 8 bit 가 호스트. 256 개 주소 (실제 254 개 사용).

## IPv6

**128bit**, `2001:0db8:85a3:0000:0000:8a2e:0370:7334` (연속 0 은 `::` 로 축약).

- 주소 수: 2^128 ≈ 3.4 × 10³⁸ 개
- 서브넷: /64 가 기본 (LAN 하나)
- 특별 주소: `::1` (loopback), `fe80::/10` (link-local), `fc00::/7` (unique local)

## Public vs Private

**Public IP**: 인터넷에서 라우팅 가능. ISP 가 할당.
**Private IP**: 로컬 네트워크. NAT 로 public 과 매핑 ([[network-nat|자세히]]).

## DHCP

동적 IP 할당 프로토콜. 클라이언트가 요청하면 DHCP 서버가 임차 (lease).
