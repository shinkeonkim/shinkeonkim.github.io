## 정의

**동적 계획법 (Dynamic Programming, DP)** 은 **큰 문제를 작은 부분 문제로 나누고**, 각 부분 문제의 **최적해를 저장**하여 **중복 계산을 제거**하는 최적화 기법. Richard Bellman 이 1950년대 고안.

분할 정복 (Divide and Conquer) 과 다른 점: 부분 문제가 **중복**됨 → 메모이제이션 또는 타뷸레이션으로 해결.

## 문제 상황과 동기

피보나치 수 `F(n) = F(n-1) + F(n-2)` 를 재귀로 구현하면:

```
F(5) 호출 → F(4) + F(3)
  F(4) → F(3) + F(2)
  F(3) → F(2) + F(1)
  ...
```

F(3) 이 **여러 번 중복 계산**. naive 재귀는 O(2^N), DP 는 **O(N)**.

핵심 통찰: *한 번 계산한 F(k) 를 저장하면 다시 안 풀어도 됨*.

PS: 대부분 최적화 / 경우의 수 문제 (knapsack, LIS, LCS, TSP, 구간 DP, 확률 DP, ...). 실무: 강화 학습 (Bellman equation), 최단 경로 (Dijkstra / Floyd-Warshall), 자원 할당.

## 핵심 아이디어

### 1. 최적 부분 구조 (Optimal Substructure)

전체 문제의 최적해가 **부분 문제의 최적해로 구성**됨.

예: 최단 경로 `A → C` 가 `A → B → C` 라면, `A → B` 도 최단.

### 2. 중복 부분 문제 (Overlapping Subproblems)

같은 부분 문제가 **여러 번 등장**.

예: F(5) 계산 중 F(3) 을 여러 번 호출.

### 3. 상태 정의 + 점화식

`dp[i]` = 상태 i 에서의 최적값 / 경우의 수.

점화식: `dp[i]` 를 이전 상태 `dp[j]` (j < i) 로 표현.

## 알고리즘

```text
DP 설계 4 단계

1. 상태 정의
   dp[i] = "i 에서 무엇을 의미하는가?"
   (예: dp[i] = i 번째까지 최소 비용)

2. 초기값
   dp[0] 또는 dp[1] 등 base case 설정.

3. 점화식
   dp[i] = f(dp[j], ...) for j < i

4. 계산 순서
   - top-down (memoization): 재귀 + 캐시
   - bottom-up (tabulation): 반복문으로 dp[0] → dp[N]
```

## 복잡도

| 항목 | 값 |
|:---|:---|
| **시간 (naive 재귀)** | O(2^N) ~ O(k^N) (지수) |
| **시간 (DP)** | O(상태 개수 × 전이 비용) |
| **공간** | O(상태 개수) 또는 O(재귀 깊이) |
| **예: Fibonacci** | O(N) 시간, O(N) 공간 |
| **예: 0-1 Knapsack** | O(N × W) 시간, O(N × W) 공간 |

## 변형 / 활용

### 1. memoization vs tabulation

| 방식 | 설명 | 장점 | 단점 |
|:---|:---|:---|:---|
| **top-down (재귀 + 메모)** | 재귀 호출 시 결과 캐싱 | 직관적, 필요한 상태만 계산 | 재귀 스택 오버플로우 위험 |
| **bottom-up (반복)** | 작은 상태부터 순서대로 계산 | 스택 안전, 빠름 | 모든 상태 계산 (불필요한 것 포함) |

### 2. 공간 최적화

dp[i] 가 dp[i-1], dp[i-2] 만 참조하면 **rolling array** (크기 2~3 배열) 로 O(1) 공간.

```cpp
long long a = 0, b = 1;
for (int i = 2; i <= n; i++) {
    long long c = a + b;
    a = b; b = c;
}
// b = fib(n)
```

### 3. 대표 DP 유형

| 유형 | 상태 | 점화식 예 |
|:---|:---|:---|
| **선형 DP** | dp[i] | dp[i] = max(dp[i-1], dp[i-2] + a[i]) |
| **배낭 DP** | dp[i][w] | dp[i][w] = max(dp[i-1][w], dp[i-1][w-wi] + vi) |
| **구간 DP** | dp[l][r] | dp[l][r] = min(dp[l][k] + dp[k+1][r] + cost) |
| **트리 DP** | dp[node][state] | dp[u] = f(dp[v1], dp[v2], ...) for children v |
| **비트마스크 DP** | dp[mask] | dp[mask \| (1<<i)] = ... |
| **확률 DP** | dp[i] = P(event at i) | dp[i] = Σ dp[j] × transition_prob |

### 4. 최적화 기법

- **Convex Hull Trick (CHT)**: 점화식이 1차 함수 최솟값 형태 → O(N log N) 또는 O(N).
- **Divide and Conquer Optimization**: dp[i][j] = min(dp[i-1][k] + cost(k, j)) 에서 최적 k 가 단조 → O(K N log N).
- **Knuth Optimization**: 구간 DP 에서 최적 분할점 단조성 → O(N^2) 대신 O(N^2).
- **Slope Trick**: 볼록 함수 유지 → 우선순위 큐로 O(N log N).

## 함정

### 1. 상태 정의 모호

"dp[i] 가 무엇을 의미하는가?" 를 명확히 해야 점화식 유도 가능. 막연히 "i 까지의 답" 은 불충분.

### 2. 초기값 실수

dp[0] 또는 dp[1] 을 잘못 설정하면 전체 답 틀림. 예: 0-1 knapsack 에서 dp[0][0] = 0, 나머지 -∞.

### 3. 계산 순서

bottom-up 에서 `dp[i]` 를 `dp[i+1]` 로 채우려 하면 참조 에러. 반드시 작은 쪽 → 큰 쪽.

### 4. 재귀 깊이

Python / Java 에서 재귀 깊이 제한 (Python 기본 1000, Java 스택 크기). top-down DP 는 `sys.setrecursionlimit(10**6)` 또는 bottom-up 전환.

### 5. 오버플로우

dp[i] 가 매우 커지면 long long 또는 모듈로 연산.
