## 정의

**문자열 (String)** 은 문자(character)의 순서 있는 나열. 프로그래밍에서는 **연속된 메모리에 저장된 문자 배열** 로 구현되며, 각 문자는 ASCII(7비트) 또는 Unicode(가변 길이) 인코딩으로 표현.

C++ `std::string`, Python `str`, Java `String` 모두 내부적으로는 배열이지만 편의 메서드(concat, substr, find 등)를 제공.

## 문제 상황과 동기

문자열은 PS 에서 **패턴 매칭, 전처리, 변환** 문제로 자주 등장.

- **naive 패턴 매칭**: text 길이 N, pattern 길이 M 일 때 O(N×M). 10^5 규모면 TLE.
- **[[KMP|KMP]]**, **[[Trie|트라이]]** 같은 전용 자료구조: O(N+M) 또는 O(M) 전처리 후 O(1) 쿼리.

핵심 통찰: 문자열은 *순서 고정, 중복 문자 가능, immutable(언어 따라 다름)* 이므로 일반 배열과 다르게 취급. 문자 간 의존성(접두사, 접미사, 팰린드롬)이 패턴을 만든다.

## 인코딩

| 인코딩 | 범위 | 비고 |
|:---|:---|:---|
| **ASCII** | 0~127 (7비트) | 영문, 숫자, 기호만. `'A'=65`, `'a'=97` |
| **Extended ASCII** | 0~255 (8비트) | 서유럽 문자 일부 |
| **UTF-8** | 가변 (1~4바이트) | Unicode 호환, 전 세계 문자 지원 |
| **UTF-16** | 가변 (2 or 4바이트) | Java `char` (BMP만 2바이트) |
| **UTF-32** | 고정 4바이트 | 모든 Unicode codepoint |

PS 에서 대부분 ASCII 범위. 한글/중국어/이모지는 UTF-8.

## 문자열 기본 연산

### 1. 길이 (length)

- C++: `s.size()`, `s.length()` - O(1)
- Python: `len(s)` - O(1)
- Java: `s.length()` - O(1)

### 2. 인덱싱 (indexing)

- **0-indexed**: `s[0]` 이 첫 문자. C++, Python, Java 모두.
- Python 은 음수 인덱스 지원 (`s[-1]` = 마지막 문자).

### 3. 부분 문자열 (substring)

- C++: `s.substr(pos, len)`
- Python: `s[start:end]` (slicing)
- Java: `s.substring(beginIndex, endIndex)` (endIndex exclusive)

복잡도: O(길이). 새 문자열 생성하므로 메모리 O(길이).

### 4. 연결 (concatenation)

- C++: `s1 + s2` (새 string 생성, O(N+M))
- Python: `s1 + s2` (immutable, 새 객체)
- Java: `s1 + s2` (새 String, `StringBuilder` 로 O(N) 누적 가능)

**함정**: 반복문 안에서 `s += ch` 는 O(N²). `StringBuilder` (Java) 또는 `vector<char>` (C++) 사용.

### 5. 비교 (comparison)

- 사전순 (lexicographical): `<`, `>`, `==` 모두 지원. 내부적으로 문자 코드 순서대로 비교.
- C++: `s1 < s2` → O(min(N,M))
- Python: `s1 < s2` → O(min(N,M))

### 6. 검색 (find)

- C++: `s.find(substr)` → `size_t` (못 찾으면 `string::npos`)
- Python: `s.find(substr)` → `int` (못 찾으면 `-1`), `s.index(substr)` → ValueError
- Java: `s.indexOf(substr)` → `int` (못 찾으면 `-1`)

**naive 구현**: O(N×M). **[[KMP|KMP]]** 로 O(N+M).

## 문자열 알고리즘 분류

| 알고리즘 | 문제 | 복잡도 | 비고 |
|:---|:---|:---|:---|
| **naive 검색** | 패턴 매칭 | O(N×M) | 짧은 패턴 / 작은 입력 |
| **[[KMP\|KMP]]** | 패턴 매칭 | O(N+M) | failure function π |
| **[[Trie\|트라이]]** | 다중 패턴, autocomplete | O(M) 검색 | 전처리 O(총 길이) |
| **Rabin-Karp** | 다중 패턴 (rolling hash) | O(N+M) avg | 충돌 주의 |
| **Aho-Corasick** | 다중 패턴 일괄 검색 | O(N + M + 출현 수) | KMP + Trie |
| **Manacher** | 팰린드롬 부분문자열 | O(N) | 중심 확장 + amortization |
| **Z-algorithm** | 접두사 매칭 배열 | O(N) | KMP 변형 |
| **Suffix Array** | 접미사 정렬 | O(N log N) | DC3, SA-IS |

## 패턴 매칭 개요

**문제**: text `T[0..N-1]` 에서 pattern `P[0..M-1]` 의 모든 출현 위치 찾기.

### naive 접근

```text
for i = 0 .. N-M:
    match = true
    for j = 0 .. M-1:
        if T[i+j] != P[j]:
            match = false
            break
    if match: report i
```

복잡도: O(N×M). worst-case: `T = "aaaa...a"`, `P = "aaab"` 처럼 매번 끝까지 비교.

### KMP 핵심

*실패 시 이미 매치된 접두사를 재사용*. failure function `π[j]` = `P[0..j]` 의 최대 proper prefix = suffix 길이. mismatch 시 `j ← π[j-1]` 로 점프, `i` 는 불변.

→ 각 문자를 최대 1회만 비교, O(N+M).

[[KMP|KMP 상세 알고리즘 참고]]

### Trie 핵심

여러 패턴을 **트리** 로 저장. 루트부터 각 문자를 간선으로. 공통 접두사는 경로 공유.

- 삽입 O(M)
- 검색 O(M)
- autocomplete: 접두사 노드 도달 후 서브트리 DFS

[[Trie|트라이 상세 알고리즘 참고]]

## 변형 / 활용

### 1. 회문 (Palindrome)

앞뒤가 같은 문자열. naive: O(N²) (모든 구간 체크). **Manacher**: O(N) (중심 확장 + amortization).

### 2. Anagram

문자 집합이 같은 문자열. 정렬 후 비교: O(N log N). 문자 빈도 배열: O(N) + O(26) (알파벳).

### 3. Longest Common Substring

두 문자열의 가장 긴 공통 부분문자열. DP: O(N×M). Suffix Array: O(N log N).

### 4. Edit Distance (Levenshtein)

삽입/삭제/치환 최소 횟수. DP: O(N×M).

### 5. Rolling Hash

부분문자열을 정수 해시로 변환. 구간 비교 O(1). Rabin-Karp 에서 사용.

## 함정

### 1. immutability 간과

Python `str`, Java `String` 은 immutable. 반복 concat `s += ch` 는 O(N²). `list` (Python) 또는 `StringBuilder` (Java) 사용.

### 2. 인덱스 범위

`s.substr(pos, len)` (C++) vs `s[start:end]` (Python) vs `s.substring(start, end)` (Java). end 는 exclusive / inclusive 차이.

### 3. find 반환값

C++ `string::npos` (매우 큰 값, `-1` 과 혼동 금지) vs Python `-1` vs Java `-1`.

### 4. 인코딩 혼동

`len(s)` 는 문자 개수가 아니라 **바이트 개수** (Python 3 은 codepoint 개수). 이모지 / 한글은 여러 바이트.

### 5. naive 패턴 매칭 TLE

N, M ≤ 10^5 면 O(N×M) = 10^10 → 10초. [[KMP|KMP]] / [[Trie|Trie]] 필수.
